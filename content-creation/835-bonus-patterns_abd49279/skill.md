## What is a Pattern Library?

A pattern library documents:
- **Winning patterns** - What worked and why
- **Failing patterns** - What didn't work and why
- **Persona-specific patterns** - What resonates with each audience
- **Channel-specific patterns** - What works on each platform
- **Seasonal patterns** - What works at different times

## Pattern Library Template

```markdown
# Marketing Pattern Library

## How to Use
- Review before each campaign
- Update after each campaign
- Track performance metrics
- Note context for each pattern

## Headline Patterns

### High-Performers (2x+ average)
| Pattern | Example | Performance | Context |
|---------|---------|-------------|---------|
| How [Persona] can [Benefit] | "How remote managers save 10 hrs/week" | 2.3x CTR | Works best for Solo Sam |

| The [Adjective] guide to [Topic] | "The complete guide to remote productivity" | 1.8x CTR | Best for pillar content |

### Underperformers (Avoid)
| Pattern | Why It Failed |
|---------|---------------|
| "Revolutionary" anything | Too hyperbolic, low trust |
| Clickbait questions | High opens, low engagement |
```

## Using Commands to Build Patterns

After each campaign, use:

```

```

Then document:
1. What content performed best?
2. What headlines got highest CTR?
3. What CTAs drove most conversions?
4. What times/days worked best?

## Pattern Library Best Practices

1. **Review before campaigns** - Check what worked before
2. **Update after campaigns** - Add new learnings
3. **Include failures** - Know what to avoid
4. **Add context** - Why it worked matters
5. **Track metrics** - Quantify performance

## CRO Patterns

### Landing Page Patterns

| Element | Winning Pattern | Improvement |
|---------|-----------------|-------------|
| Headline | Specific benefit + number | +40% engagement |
| Subheadline | "How" or "Who it's for" | +25% clarity |
| CTA | First-person ("Start My Trial") | +28% clicks |
| Form fields | 3-4 fields max | +35% completions |
| Social proof | Logos + "Join X teams" | +22% trust |

### Form Patterns

| Pattern | Impact |
|---------|--------|
| Single column layout | +15% completion |
| Progress indicator (multi-step) | +20% completion |
| Inline validation | +22% reduction in errors |
| Smart defaults | +10% completion |
| "No credit card required" | +18% signups |

### Popup Patterns

| Type | Timing | Conversion |
|------|--------|------------|
| Exit intent | Mouse leaves viewport | 3-5% capture |
| Scroll 50% | Engaged visitors | 2-4% capture |
| Time delay 30s | After reading | 1-3% capture |

**High-Converting Popup Elements:**
- Clear value in headline (not "Subscribe")
- Single field (email only)
- Visible close button
- Mobile-friendly (full-screen OK)
- Frequency cap (once per session)

### Signup Flow Patterns

| Pattern | Conversion Impact |
|---------|-------------------|
| Email-only start | +45% initial capture |
| Google/SSO option | +30% for consumer |
| Progressive profiling | +25% vs long form |
| Show password option | +12% completions |

### Onboarding Patterns

| Element | Activation Impact |
|---------|-------------------|
| Empty state with value | +35% action rate |
| Sample data/templates | +28% exploration |
| 3-step checklist | +40% completion |
| Aha moment in <5 min | +50% trial conversion |

### Paywall Patterns

| Trigger | Conversion |
|---------|------------|
| After value moment | Highest (2-4%) |
| Feature gate | Medium (1-2%) |
| Usage limit | Variable |
| Trial expiry | Depends on engagement |

**Effective Upgrade Screen Elements:**
- Feature comparison table
- Current plan highlighted
- Testimonial from similar user
- "Most popular" badge
- Annual discount prominent
